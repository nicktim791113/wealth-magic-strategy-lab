# 致富魔法短線策略實驗室 MVP 規格

## 目標

建立一個每天可重複使用的投資研究環境，把市場資訊轉成可驗證假設，再用模擬交易與復盤資料累積策略勝率、期望值、來源可靠性與失敗原因。第一版不自動下單，先完成「資訊 -> 假設 -> 驗證 -> 迭代」閉環。

## A. 系統頁面設計

1. 今日市場
   - 顯示台股、美股、費半、VIX、匯率等市場風向。
   - 顯示候選標的、題材強弱、假設看板、策略排行、事件反應曲線、風控上限。

2. 假設實驗
   - 每日建立 3-10 個假設。
   - 每個假設必須有標的、策略版本、訊號來源、論點、進場條件、出場條件、撤退條件、驗證天數。
   - 狀態包含 watching、triggered、won、lost、expired。

3. 策略排行榜
   - 依策略樣本數、勝率、賺賠比、期望值、最大回撤、錯誤訊號率計算總分。
   - 樣本數不足的策略只能觀察或模擬。

4. 模擬交易
   - 由已建立假設轉成紙上交易。
   - 每筆交易必須有進場、目標、停損、計畫風險、結果與備註。

5. 復盤中心
   - 每日盤前、盤中、盤後、週檢討流程。
   - 錯誤分類為資訊錯、時機錯、價格錯、紀律錯、環境錯。

6. 資料庫
   - 顯示 MVP 資料欄位與未來 SQLite/PostgreSQL 轉換方向。

7. 情報來源
   - 顯示每日自動抓取的行情與新聞來源狀態。
   - 允許貼上新聞、影片逐字稿、法說摘要、社群觀點或手動筆記。
   - 將資訊轉成假設草稿，並保留來源可信度、週期、題材、影響標的。

## B. 資料庫欄位設計

核心資料表：

- `market_sessions`: 每日市場狀態。
- `signals`: 消息、技術、籌碼、財報、題材訊號。
- `hypotheses`: 每日短線假設。
- `strategy_versions`: 策略版本與統計。
- `simulated_trades`: 模擬交易。
- `reviews`: 每日復盤。
- `score_snapshots`: 每日策略分數快照。
- `intelligence_sources`: 新聞、影片、公告與手動情報來源。
- `source_outcomes`: 來源轉成假設後的驗證結果，用來評估資料來源可靠性。

完整 SQL 草案放在 `schema/short_term_strategy_lab.sql`。

## C. 每日操作流程

盤前：

1. 更新台股、美股、費半、VIX、美元台幣與期貨方向。
2. 產生候選清單，排除量能不足與消息不明標的。
3. 建立今日 3-10 個可驗證假設。
4. 每個假設寫清楚進場、出場、撤退條件。

盤中：

1. 只在觸發條件成立時建立模擬交易。
2. 沒觸發就不做。
3. 監控停損、目標、族群強弱與錯誤訊號。

盤後：

1. 標記假設成立、失敗、過期或繼續觀察。
2. 更新策略統計。
3. 記錄錯誤原因。

週檢討：

1. 樣本數滿 30 筆後才評估升級。
2. 低期望值策略降級。
3. 市場狀態改變時建立新版本重新測試。

## D. 策略評分公式

基礎公式：

```text
勝率 = 成功次數 / 樣本數
失敗率 = 1 - 勝率
賺賠比 = 平均獲利 / 平均虧損
期望值 = 勝率 * 平均獲利 - 失敗率 * 平均虧損
樣本信心 = min(樣本數 / 50, 1)
風險扣分 = 最大回撤 * 0.42 + 錯誤訊號率 * 8
總分 = (期望值 * 14 + 勝率 * 34 + 賺賠比 * 12) * 樣本信心 - 風險扣分
```

升級門檻：

- 樣本數 >= 30。
- 期望值 > 0。
- 最大回撤低於策略風控門檻。
- 策略在當前市場狀態仍有效。

降級門檻：

- 連續 10 筆期望值轉負。
- 錯誤訊號率明顯升高。
- 市場 regime 改變且舊規則失效。

## E. MVP 功能

已落地：

- React + Vite 工作台。
- 今日市場總覽。
- `npm run refresh:data` 每日抓取公開行情與 RSS 新聞，輸出 `public/data/market-lab-latest.json`。
- 候選清單優先使用每日刷新資料，失敗時回到示範資料。
- 候選標的一鍵建立假設，支援短線、波段、長線。
- 情報來源頁，可貼上新聞或影片摘要並生成假設草稿。
- 假設建立、狀態更新與 localStorage 保存。
- 假設轉模擬交易。
- 模擬交易結果標記。
- 策略排行榜與評分公式。
- 每日流程勾選與復盤筆記。
- 資料庫欄位檢視。

下一階段：

- 加入更多穩定資料源與 API key 設定，例如 Finnhub、Alpha Vantage、NewsAPI、券商 API。
- 金融影片逐字稿自動擷取與摘要。
- 回測引擎。
- SQLite API 後端。
- 策略版本 diff。
- 自動產生每日盤前候選清單。
- 真實下單前風控檢查，不直接自動下單。

## 資料更新指令

每天盤前先執行：

```powershell
npm run refresh:data
npm run dev
```

或直接：

```powershell
npm run dev:daily
```

第一版公開資料來源：

- TWSE OpenAPI：台股上市日成交資訊。
- Yahoo Finance chart endpoint：美股、ETF、ADR 與部分台股 chart 資料。
- SEC RSS：官方 SEC 新聞/公告 RSS。
- Google News RSS：AI、半導體、台股 AI 伺服器、資料中心電力、高股息等主題新聞。

Yahoo Finance chart 與 Google News RSS 是免金鑰公開端點，穩定性不等同付費正式 API；因此系統會保留來源狀態與 fallback。未來若要提高可靠度，應接入正式資料供應商或券商 API。
